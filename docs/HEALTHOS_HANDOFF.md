# Полный технический Handoff проекта HealthOS

Этот документ является точкой входа для Codex. Проект был проанализирован, запущен и задокументирован 10 июля 2026 года.

## 1. Где находится проект
- **GitHub-репозиторий**: [https://github.com/VartD/healthos-agent](https://github.com/VartD/healthos-agent)
- **Владелец и название**: `VartD/healthos-agent`
- **Доступ**: Public
- **Основная ветка**: `main`
- **Актуальная рабочая ветка**: `manus-handoff-2026-07-10` (создана для передачи Codex)
- **SHA последнего коммита в main**: `39a54fe`
- **Локальные изменения**: Нет. Все локальные файлы соответствуют GitHub. Ветка `manus-handoff-2026-07-10` содержит актуальный код.
- **Расположение компонентов**:
  - Frontend, Mini App, мобильное приложение: **Отсутствуют**.
  - Backend: Папка `backend/`.
  - Telegram-бот: Папка `bot/`.

## 2. Архитектура
Подробное описание в [ARCHITECTURE.md](ARCHITECTURE.md). Проект представляет собой монолитный MVP на FastAPI и python-telegram-bot. Заявленные в описании технологии (Aiogram, React Native, GPT-4.1-mini, APScheduler) в коде отсутствуют.

## 3. Реальный статус функций
Подробная таблица в [STATUS.md](STATUS.md). Базовый ввод данных работает, но сложная аналитика, дашборды и проактивные уведомления не реализованы.

## 4. AI-Агенты
В коде нет ни одного AI-агента. Логика реализована через детерминированные if/else правила. Подробнее в [STATUS.md](STATUS.md).

## 5. База данных
Используется PostgreSQL (или SQLite для локальных тестов) с единственной таблицей `health_events`. Миграций нет. Схема описана в [ARCHITECTURE.md](ARCHITECTURE.md).

## 6. Инфраструктура и деплой
Присутствует `docker-compose.yml` для локального запуска БД и backend. Скриптов деплоя, CI/CD и конфигурации Cloudflare/Tor в репозитории нет.

## 7. Воспроизведение проекта
Инструкция по запуску в чистом окружении описана в [RUNBOOK.md](RUNBOOK.md).

## 8. Тестирование
**Тестов нет.** Ни unit, ни integration, ни E2E тесты в репозитории не обнаружены.

## 9. Безопасность
Открытый API без авторизации. Подробности в [DECISIONS.md](DECISIONS.md).

## 10. Незавершённые задачи (Backlog)
Описаны в [BACKLOG.md](BACKLOG.md). Первоочередная задача — рефакторинг на асинхронный код и внедрение реальных LLM-агентов.

## 11. Переданные артефакты
Все документы находятся в папке `docs/` в ветке `manus-handoff-2026-07-10`.
