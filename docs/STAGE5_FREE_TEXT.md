# Stage 5 — свободный ввод, первый вертикальный срез

## Цель

Пользователь пишет явное событие обычным русским текстом. Бот распознаёт его,
передаёт в существующий защищённый backend, сохраняет в журнал и отвечает
коротким подтверждением. Slash-команды остаются fallback-интерфейсом.

## Поддержано

- вода с объёмом;
- глюкоза и мочевая кислота;
- давление `верхнее/нижнее` и опциональный пульс;
- еда, тренировка и симптомы;
- кофе/чай только с явно указанным объёмом;
- сауна с длительностью;
- явно обозначенные лекарства и добавки.

## Safety и границы

- parser консервативный: при неоднозначности событие не сохраняется;
- все значения повторно валидирует backend;
- parser не ставит диагнозы и не меняет назначения;
- давление ≥180 и/или ≥120 мм рт. ст. всегда проходит через детерминированный
  red flag; LLM не может его отменить;
- при таком давлении бот просит повторить измерение через минуту и связаться с
  врачом, если значение сохраняется; при тревожных симптомах сообщает звонить 112;
- порог и последовательность действий основаны на памятке American Heart
  Association: <https://www.heart.org/en/health-topics/high-blood-pressure/understanding-blood-pressure-readings>.

## Acceptance

```bash
pytest
```

Ожидаемый результат: 34 теста. Отдельно покрыты parser, неоднозначный ввод,
запись через настоящий FastAPI application и критическое давление с симптомами
и без них.

## Не входит в этот срез

Фото, голос, LLM-интерпретация, multi-event сообщения, состояние диалога,
подтверждение событий средней уверенности и свободный утренний sleep check-in.
