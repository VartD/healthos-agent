# Незавершённые задачи (Backlog)

Проект находится на стадии раннего прототипа (MVP). Заявленный функционал реализован лишь на малую долю.

## Технический долг и временные решения
- **Отсутствие миграций БД (P1)**: Используется `Base.metadata.create_all`. Необходимо внедрить Alembic.
- **Синхронный API (P1)**: FastAPI используется с синхронными функциями, блокирующими event loop. Необходимо переписать на `async def` и использовать `asyncpg` / `aiosqlite`.
- **Синхронный Telegram Bot (P1)**: Бот делает синхронные HTTP-запросы через `httpx.AsyncClient` (используется некорректно, функции `_post_event` и `_analyze` асинхронные, но архитектура бота требует рефакторинга).
- **Hardcoded правила (P2)**: Все медицинские пороги (глюкоза > 7, мочевая кислота > 360) зашиты прямо в код (`state_engine.py`).
- **Отсутствие тестов (P0)**: В репозитории нет ни одного файла с тестами.

## Запланировано, но не начато (Блокеры для полноценного продукта)
- **AI-Агенты (P0)**: Интеграция с LLM (GPT-4.1-mini) для анализа симптомов и генерации рекомендаций.
- **APScheduler (P0)**: Реализация утренних и вечерних чекинов, напоминаний о приеме лекарств.
- **Telegram Mini App (P1)**: Разработка frontend-части для дашбордов и трендов.
- **Мобильное приложение (P2)**: React Native / Expo клиент.
- **Обработка фото (P1)**: Интеграция Vision API для распознавания показателей с Apple Watch.

## Что Codex должен сделать первым
1. **Рефакторинг API и БД**: Перевести FastAPI и SQLAlchemy на асинхронный режим, добавить Alembic.
2. **Интеграция LLM**: Заменить жестко закодированные правила в `engine` на вызовы LLM (langchain/openai) для реализации первого AI-агента (например, GP).
3. **Настройка планировщика**: Добавить APScheduler для проактивных уведомлений.
