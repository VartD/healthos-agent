# Архитектурные решения и безопасность

## Безопасность

### Найденные проблемы
1. **Пользовательская авторизация отсутствует (Критичность: High)**
   Codex добавил обязательный сервисный `X-API-Key`, поэтому публичное анонимное
   чтение и запись закрыты. Однако API-key аутентифицирует доверенный клиент, а не
   конечного пользователя. До внешнего пилота нужны user sessions и ownership checks.
2. **Идентификация пользователя (Критичность: Medium)**
   Бот использует `chat.id` из Telegram напрямую как `user_id`. Нет проверки, зарегистрирован ли пользователь.
3. **Валидация данных (остаточная критичность: Medium)**
   Codex добавил базовые bounds и обязательные поля: `/water -500` и `/glucose 9999`
   отклоняются. Ещё нужны канонические единицы, схемы давления и лабораторных данных,
   provenance и подтверждение автоматически распознанных значений.
4. **CORS и Rate Limiting (Критичность: Low)**
   Отсутствуют. FastAPI поднят без CORSMiddleware и без ограничений на количество запросов.
5. **Шифрование данных (Критичность: Medium)**
   Медицинские данные хранятся в открытом виде в базе данных PostgreSQL/SQLite.
6. **SQL Injection**
   Уязвимостей не найдено. Используется SQLAlchemy ORM, сырые запросы отсутствуют.

### Секреты
Секретов в истории Git не найдено. В файле `.env.example` находятся только заглушки.

## Принятые решения при Handoff
1. **Локализация**: Проект был найден в публичном репозитории GitHub (`VartD/healthos-agent`). Код значительно отличается от заявленного в задании (отсутствуют AI-агенты, Aiogram, React Native, 15 endpoints).
2. **Поиск HealthOS 2.0**: Был проведен исчерпывающий поиск по всей файловой системе sandbox, архивам, внутренним каталогам Manus и всем репозиториям аккаунта VartD. Код с 4498 строками, 14 агентами и Mini App **не найден**. Предыдущий отчёт не подтверждался фактической кодовой базой и, вероятно, был построен на основе планов или описаний, а не реальных файлов.
3. **Ветка handoff**: Создана ветка `manus-handoff-2026-07-10` для фиксации текущего достоверного состояния.
4. **Тестовая БД**: Добавлено правило в `.gitignore` для исключения локальных баз SQLite (`*.db`).
5. **AI safety**: детерминированные red flags сохраняются. LLM в будущем используется
   для оркестрации и объяснения, но не заменяет safety rules.
